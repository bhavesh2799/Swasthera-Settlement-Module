## CONTEXT

You are enhancing a B2B brand onboarding and settlement compliance platform called **Swasthera**.
The stack is React on the frontend and Node.js/Express on the backend. Use the folder structure,
naming conventions, and patterns already present in the project. Where no pattern exists,
default to: React functional components with hooks, Express Router for API grouping,
and a RESTful API design.

Apply all changes incrementally — do not rewrite files unless necessary. Add new routes,
components, and services as separate files and import them into existing entry points.

## ROLE-BASED ACCESS CONTROL (FOUNDATION — BUILD THIS FIRST)

### Roles
- `maker` — creates and submits onboarding drafts; views own submissions
- `checker` — Finance role; approves/rejects/requests edits; accesses compliance reports
- `admin` — manages users; configures global TDS/TCS rates; views all modules

### Backend
Create `middleware/rbac.js`:
```js
// Usage: router.get('/route', authenticate, authorize(['checker', 'admin']), handler)
```
- `authenticate` middleware: verify JWT from `Authorization: Bearer <token>` header
- `authorize(roles)` middleware: check `req.user.role` against allowed roles array
- Attach `req.user = { id, role, name }` after token verification

### Frontend
Create `context/AuthContext.jsx`:
- Store `{ user, token, login, logout }` in context
- `useAuth()` hook for component access
- `<ProtectedRoute roles={['checker']} />` wrapper component that redirects if role not allowed
- Show/hide UI elements based on `user.role` — never rely on frontend role checks for data security

**Mandatory field validation per entity type** — create `services/validationRules.js`:
```js
const mandatoryFields = {
  private_limited:  ['pan', 'gstn', 'tan', 'cin'],
  public_limited:   ['pan', 'gstn', 'tan', 'cin'],
  llp:              ['pan', 'gstn', 'tan', 'cin', 'llp_code'],
  partnership:      ['pan', 'gstn', 'tan'],
  proprietorship:   ['pan', 'tan'],  // cin optional, gstn optional with toggle
};
```

**IFSC lookup** — `GET /api/utils/ifsc/:code`
- Call `https://ifsc.razorpay.com/:ifscCode` (or equivalent)
- Return `{ bank, branch, city, state }` for auto-populate

**GST auto-fetch** — `POST /api/utils/gst-lookup`
- Accept `{ gstn }`, call GST verification API
- Return `{ registered_address, trade_name, status }`

**KYB service** — `services/kybService.js`
- Cross-validate CIN ↔ PAN ↔ GST via third-party API
- On success: update company `kyb_verified: true, kyb_verified_at`
- On failure: return `{ mismatched_fields: [] }`

**Auto-save** — all `POST /api/onboarding/*` routes should upsert (not 409 on re-save).
Return the full updated draft on every save.

## SHARED INFRASTRUCTURE

### Auto-save Pattern (Frontend)
Create `hooks/useAutoSave.js`:
```js
// Debounce form state → call API every 2 seconds on change
// Show "Saving..." / "Saved ✓" / "Save failed ✗" indicator in form header
// On unmount: flush pending save immediately
```

### Audit Log Middleware (Backend)
Create `middleware/auditLogger.js`:
```js
// After any mutating route (POST/PUT/DELETE) on onboarding, commercials, settlements:
// Write to AuditLog: { entity_type, entity_id, action, changed_by, changed_fields, timestamp }
// Attach as: router.post('/route', authenticate, auditLogger('Company', 'update'), handler)
```

### Document Storage
- Use `multer` for multipart upload handling
- Store in `/uploads/` locally in Replit (or swap to S3-compatible if configured)
- Return signed URL or direct path for download
- Validate: file type (pdf/jpg/png), max size 10MB

### Error Response Standard
All API errors return:
```json
{ "success": false, "error": { "code": "VALIDATION_ERROR", "message": "...", "fields": {} } }
```
All success responses return:
```json
{ "success": true, "data": { ... } }
```

## NOTES FOR REPLIT AGENT

- Do not install unnecessary packages. Prefer packages already in package.json.
- For PDF generation use `pdfkit` (backend) or `html2canvas + jspdf` (frontend).
- For date math use `date-fns` (already common) — avoid moment.js.
- Keep all business logic (commission calc, TDS reversal eligibility, settlement calc)
  in `services/` files — not in route handlers — so they are unit-testable.
- All monetary values: store as integers (paise/cents), display as formatted currency.
- All dates: store as UTC ISO strings, display in IST (Asia/Kolkata, UTC+5:30).
