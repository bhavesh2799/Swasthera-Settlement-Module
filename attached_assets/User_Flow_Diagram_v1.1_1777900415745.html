Exception & reversal paths

Branches off the happy path that the canvas above doesn't draw inline. Each one fires from a state in Phase 03 / 04 and routes back into the ledger.
★ return_bag_delivered
Return accepted
Bag PERMANENTLY excluded. Commission + TCS + TDS reversed in current month's ledger. Never re-eligible in any future cycle.
return_request_cancelled
Hold lifted
Settlement HOLD removed. Bag returns to eligible pool for the next cycle. No ledger change.
cancelled_* (post-invoice)
Cancellation reversal
Bag excluded. If cancellation arrived after bag_invoiced, TCS + TDS are reversed before next month's 7th / 10th deadline.
rto_bag_delivered
RTO loss
Bag excluded. TCS + TDS reversed. Tagged as RTO loss for the warehouse state's TCS filing in that month.
partially_delivered
Split eligibility
Only delivered qty enters cycle. TCS + TDS are recomputed on delivered qty. Returned qty follows the return path independently.
cycle spans months
Cross-month reversal
Pre-deadline → reversal nets against same-month liability. Post-payment → next compliance cycle adjustment.
post-settlement return
Credit note
No reverse bank transfer. System raises a credit note against the original Commission Invoice; netted off the brand's next settlement.
Finance reject
Cycle recalc
Statement returned with comments. Cycle does not progress to Phase 06 for that company until re-approved. Other companies in the cycle are independent.
